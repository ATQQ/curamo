# 技术方案实现文档

## 1. 引言

### 1.1 背景与目标
本项目旨在开发一款 **AI 内容精选助手 (Curamo)**，帮助用户从海量信息（RSS、网页等）中高效筛选、总结并生成高质量的精选内容（如日报、周刊）。系统通过 AI 技术实现自动化处理，并提供现代化的编辑与发布体验。

### 1.2 范围
本文档涵盖前端 Web 应用与后端 API 服务的设计与实现细节。

---

## 2. 总体架构

### 2.1 项目组织
采用 **Monorepo** 结构管理代码，使用 `bun` 作为统一的包管理器。

```text
curamo/
├── apps/
│   ├── web/          # 前端 Vue 3 应用
│   └── api/          # 后端 Elysia 应用
├── packages/         # 共享包 (可选)
│   └── shared/       # 通用类型定义、工具函数
├── docs/             # 项目文档
├── package.json      # Root package.json (Bun workspaces)
└── README.md
```

### 2.2 技术栈概览

| 模块 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **运行环境** | **Bun** | 高性能 JavaScript 运行时，统一前后端依赖管理与脚本执行 |
| **前端框架** | **Vue 3 + TypeScript** | 组合式 API，配合 Vite 构建 |

| **Markdown** | **markstream-vue** | 专为流式 AI 响应设计的 Markdown 渲染组件 |

| **AI 编排** | **LangChain.js** | 构建 AI 应用的核心框架 |
| **AI 协议** | **MCP (Model Context Protocol)** | 统一 AI 技能与工具调用的标准接口 |
| **AI 模型** | **智谱 AI (GLM-5)** | 核心推理模型 |
| **AI 服务** | **七牛云** | 辅助 AI 能力 |

## 3. 数据库设计 (Database Schema)

基于 Turso (SQLite)，核心表结构设计如下：

### 3.1 实体关系图 (ERD 简述)
- `Source` (1) -> (N) `Article`
- `Task` (1) -> (N) `TaskItem` -> (1) `Article`
- `Task` (1) -> (N) `Snapshot`
- `Task` (N) -> (1) `Template`

### 3.2 表结构定义

#### `sources` (数据源)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| name | TEXT | 名称 |
| url | TEXT | RSS 地址或主页链接 |
| type | TEXT | `rss` / `manual` |
| config | JSON | 配置项（如是否自动翻译、抓取频率） |
| created_at | INTEGER | 创建时间戳 |

#### `articles` (文章)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| source_id | TEXT | 外键 -> sources.id |
| title | TEXT | 标题 |
| url | TEXT | 原文链接 |
| content | TEXT | 正文内容（清洗后） |
| raw_content | TEXT | 原始抓取内容 |
| published_at | INTEGER | 发布时间 |
| ai_summary | TEXT | AI 生成的摘要 |
| tags | JSON | 标签数组 |
| category | TEXT | 分类 |
| status | TEXT | `unread`, `read`, `archived` |

#### `templates` (模板)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| name | TEXT | 模板名称 |
| content_pattern | TEXT | 模板内容，支持变量 `{{summary}}` 等 |

#### `tasks` (精选任务)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| title | TEXT | 任务标题 (e.g., "前端日报 10-27") |
| template_id | TEXT | 外键 -> templates.id |
| status | TEXT | `draft`, `generated`, `published` |
| created_at | INTEGER | 创建时间 |

#### `task_items` (任务关联文章)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| task_id | TEXT | 外键 -> tasks.id |
| article_id | TEXT | 外键 -> articles.id |
| processed_content | TEXT | 针对该任务的 AI 改写/润色内容 |
| is_selected | BOOLEAN | 是否最终入选 |
| sort_order | INTEGER | 排序权重 |

#### `snapshots` (版本历史)
| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | TEXT (UUID) | 主键 |
| task_id | TEXT | 外键 -> tasks.id |
| content | TEXT | 完整的 Markdown 内容 |
| version_note | TEXT | 版本备注 (e.g., "AI 初稿", "人工修订") |
| created_at | INTEGER | 创建时间 |

## 4. 后端设计 (Backend - apps/api)

### 4.1 核心模块
- **Server**: ElysiaJS 应用入口，配置 CORS、Swagger 等中间件。
- **Database**: Drizzle ORM 配置与 Turso 连接。
- **Queue**: 简单的内存或 Redis 队列，用于处理耗时的 RSS 抓取任务。
- **AI Agent**: 封装 LangChain 逻辑。

### 4.2 AI 模块架构 (LangChain + MCP)
使用 `langchainjsmcp` (自定义模块名) 来实现 AI 能力的模块化。

1.  **Skills as Tools (MCP Tools)**:
    -   `fetch_rss(url)`: 获取 RSS 内容。
    -   `extract_content(url)`: 提取网页正文 (使用 Firecrawl 或类似技术)。
    -   `summarize_text(text, style)`: 调用 LLM 进行摘要。
    -   `classify_article(text, categories)`: 文本分类。

2.  **Workflow (Chains)**:
    -   **Ingestion Chain**: Fetch RSS -> Parse -> Deduplicate -> Extract Content -> Save to DB.
    -   **Curation Chain**: Select Articles -> Apply Template -> Generate Draft -> Stream to Frontend.

3.  **Model Provider**:
    -   统一封装智谱 AI API 调用。
    -   使用七牛云辅助服务（如存储图片、多媒体处理）。

### 4.3 API 接口规划
采用 Elysia 的 Eden Treaty 模式，实现前后端类型共享。

-   `GET /sources`: 获取源列表
-   `POST /sources`: 添加源
-   `POST /sources/:id/sync`: 触发立即抓取
-   `GET /articles`: 获取文章列表（支持过滤）
-   `POST /tasks`: 创建任务
-   `POST /tasks/:id/generate`: 触发 AI 生成初稿 (Streaming Response)
-   `GET /tasks/:id`: 获取任务详情
-   `PUT /tasks/:id/content`: 保存编辑内容

## 5. 前端设计 (Frontend - apps/web)

### 5.1 目录结构
```text
src/
├── components/
│   ├── ui/           # 基础 UI 组件
│   ├── editor/       # 编辑器相关组件
│   └── MarkStream.vue # 流式 Markdown 渲染组件
├── stores/           # Pinia 状态管理 (useTaskStore, useArticleStore)
├── api/              # Elysia Client (Eden)
├── views/            # 页面视图
└── App.vue
```

### 5.2 关键组件实现

#### `markstream-vue`
这是一个核心自定义组件，用于处理 AI 的流式输出。
-   **Props**: `stream: ReadableStream | string`
-   **功能**:
    -   接收后端 SSE (Server-Sent Events) 或 WebSocket 数据流。
    -   实时追加文本到内部缓冲区。
    -   使用 `markdown-it` 或类似库实时渲染 HTML。
    -   支持光标跟随和打字机效果。

#### 编辑器 (Editor)
-   基于 **ProseMirror** 或 **Tiptap** 构建。
-   支持 Markdown 快捷键。
-   **AI 辅助**: 选中通过 Slash Command (`/`) 呼出 AI 菜单（润色、扩写）。

### 5.3 交互流程
1.  **数据接入**: 用户在“设置”页添加 RSS 源，前端调用 `POST /sources`。
2.  **内容筛选**: 在“素材库”页，用户浏览抓取的文章，勾选感兴趣的条目，点击“创建任务”。
3.  **AI 生成**:
    -   进入任务页，点击“生成”。
    -   前端建立 SSE 连接 `/tasks/:id/generate`。
    -   后端 LangChain 开始处理，通过 MCP 调用工具，并将结果流式返回。
    -   `markstream-vue` 实时展示生成的初稿。
4.  **人工精修**: 用户在右侧编辑器中修改内容，自动保存（Debounce `PUT` 请求）。

## 6. 环境与部署

### 6.1 环境变量 (.env)
```bash
# Backend
BUN_ENV=production
PORT=3000
DATABASE_URL=./curamo.db
AI_API_BASE=https://api.qnaigc.com/v1
AI_API_KEY=<七牛云 AI API KEY>
AI_API_MODEL=z-ai/glm-5


# Frontend
VITE_API_BASE_URL=http://localhost:3000
```

### 6.2 开发流程
1.  `bun install`: 安装所有依赖。
2.  `bun run dev`: 同时启动前端 Vite 和后端 Elysia。

### 6.3 部署方案
-   **Backend**: 打包为单一可执行文件 (`bun build --compile`) 或 Docker 容器，部署至云服务器/边缘节点。
-   **Frontend**: 构建静态资源 (`bun run build`)，部署至 Vercel/Netlify 或 Nginx 容器。
-   **Database**: 本地文件。
