# 内容精选助手 (Curamo) 项目需求文档

## 1. 项目概述 (Project Overview)

### 1.1 背景
随着信息爆炸，用户每天面临海量的内容输入。如何从繁杂的信息流（RSS、Newsletter、网页等）中高效筛选、总结并生成有价值的精选内容（如日报、周刊、专题报告），成为了一个痛点。

### 1.2 目标
开发一款基于 AI 的内容精选助手（Content Curation Assistant），帮助用户自动化订阅、筛选、总结和生成高质量的精选内容。核心通过 AI 技术（如 LangChain、LLM）实现内容的智能分类、摘要、翻译和重组，并提供灵活的编辑器和模板系统，最终输出标准化的精选内容。

### 1.3 范围
本项目包含前端 Web 应用和后端服务。
- **前端**：基于 Vue 3 + TypeScript 开发的单页应用，提供类 Chat 的交互界面。
- **后端**：基于 Bun + Elysia 开发的 API 服务，集成 LangChain 和 AI 服务（智谱 AI/七牛云）。

## 3. 功能需求 (Functional Requirements)

### 3.1 数据源与抓取 (Data Sources & Fetching)
- **RSS 源配置**：
  - 支持添加、编辑、删除 RSS 订阅源。
  - 支持对 RSS 源进行分组管理。
  - **自动翻译**：支持配置是否对特定源的内容标题/正文进行自动翻译（如英文转中文）。
- **手动录入**：
  - 支持通过输入 URL 手动添加文章。
  - 支持直接粘贴文本内容作为素材。
- **内容抓取**：
  - 系统后台定时或按需抓取 RSS 源的最新文章。
  - 解析文章的正文内容（去除广告、侧边栏等无关信息）。

### 3.2 内容处理与 AI (Content Processing & AI)
- **智能分类与标签**：
  - AI 根据文章内容自动打标签（Tags）和分类（Categories）。
  - 支持用户自定义分类体系，AI 依据体系进行归类。
- **内容摘要与提取**：
  - **单篇总结**：针对单篇文章，生成简明扼要的摘要（Summary）。
  - **多篇聚合**：针对聚合类文章（如“本周十篇好文”），AI 识别并提取其中的子项内容。
  - **多语言支持**：集成翻译能力，支持跨语言的内容处理。

### 3.3 精选任务管理 (Curation Task Management)
- **任务创建**：
  - 用户新建“精选任务”（如“2023-10-27 前端日报”）。
  - 选择目标模板（Template）。
- **文章筛选**：
  - 从已抓取的文章库中筛选素材。
  - 支持按时间、来源、分类、标签进行过滤。
  - 支持多选文章加入当前任务。
- **初稿生成**：
  - 点击“开始生成”后，AI 按照选定模板，将选中的文章处理（摘要/提取）后生成初稿。
  - 生成过程需落库，支持断点续传或重新生成。

### 3.4 编辑与生成 (Editing & Generation)
- **双栏交互界面**：
  - **左侧**：任务列表与导航。
  - **右侧**：操作面板与编辑器。
- **Markdown 编辑器**：
  - 支持 Markdown 语法编辑。
  - **实时预览**：所见即所得的预览效果。
  - **AI 辅助编辑**：在编辑器中通过对话指令（Chat）要求 AI 修改、润色、缩写或扩写特定段落。
- **版本管理**：
  - 自动保存编辑历史。
  - 支持查看历史版本并回滚。
  - 记录 AI 修改前后的对比。
- **人工筛选**：
  - 在生成最终内容前，允许用户对 AI 提取的单条内容进行勾选/取消勾选，决定是否纳入最终版。

### 3.5 系统配置与管理 (System Configuration & Management)
- **模板管理**：
  - 自定义精选内容的输出模板（如 Markdown 格式模板）。
  - 模板支持变量（如 `{{title}}`, `{{summary}}`, `{{link}}`）。
- **AI 配置**：
  - 配置 AI 模型参数（模型名称、Temperature 等）。
  - 管理 Prompt 提示词（如摘要风格、翻译指令）。
- **分类/标签管理**：
  - 维护全局的分类和标签库。

## 4. 非功能需求 (Non-Functional Requirements)

- **性能**：
  - 文章抓取与解析应在后台异步进行，不阻塞前端操作。
  - AI 生成过程需提供进度反馈（Loading 状态或流式输出）。
- **可用性**：
  - 界面需简洁美观，符合现代审美。
  - 编辑器操作流畅，无明显卡顿。
- **数据隐私**：
  - 用户配置的 API Key 等敏感信息需加密存储。
- **扩展性**：
  - 后端架构应支持未来扩展更多数据源（如 Twitter/X, Telegram 等）。

## 5. 界面与交互设计 (UI/UX Design)

### 5.1 布局结构
- **整体风格**：简洁、高效、现代（参考 Readwise Reader, Notion）。
- **主界面**：
  - **侧边栏 (Sidebar)**：显示“我的任务”、“订阅源”、“模板管理”、“设置”。
  - **内容区 (Content Area)**：
    - **任务视图**：左侧为文章待选列表（支持搜索/过滤），右侧为精选内容预览/编辑器。
    - **对话框 (Chat Interface)**：嵌入在编辑器旁或底部，用于向 AI 下达指令（如“把这段改得更幽默一点”）。

### 5.2 核心交互流程
1.  **新建任务** -> 选择模板 -> 设定筛选条件（可选）。
2.  **浏览素材** -> 在左侧列表勾选感兴趣的文章 -> 添加到“待处理区”。
3.  **AI 处理** -> 点击“生成初稿” -> AI 逐个处理文章并按模板组合。
4.  **人工微调** -> 在右侧编辑器中查看初稿 -> 修改文字/调整顺序 -> 保存版本。
5.  **导出/发布** -> 复制 Markdown 或生成分享链接。

## 6. 数据模型简述 (Data Model Draft)

- **Source (数据源)**: `id`, `name`, `url`, `type` (rss/manual), `config` (translation, etc.)
- **Article (文章)**: `id`, `source_id`, `title`, `content`, `url`, `published_at`, `status` (unread/read), `ai_summary`, `tags`, `category`
- **Task (精选任务)**: `id`, `title`, `template_id`, `created_at`, `status` (draft/published)
- **TaskItem (任务项)**: `id`, `task_id`, `article_id`, `processed_content` (AI处理后的内容), `is_selected` (是否入选)
- **Snapshot (版本快照)**: `id`, `task_id`, `content`, `created_at`, `version_note`
- **Template (模板)**: `id`, `name`, `content_pattern`
